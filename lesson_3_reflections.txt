When would you want to use a remote repository rather than keeping all your work
local?

    A remote repository is the best way to keep it safe and available to everyone whom may need work on it.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    Making things manually is better to maintain the remote repository always clean and safe from changes that isn't tested or approved to be on production.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    Branches are repositories forks used to develop a some feature, correcting bugs, trying a new experimental functionality etc.
    Clone is a command to copy a remote repository.
    Fork is the action of clone someone's repository as new one in our own repository.
